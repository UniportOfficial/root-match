## 1. 분리 사유 (Why W2-2.5 exists)

W2-2 atomic commit scope는 **Better Auth 1.6 integration + cookie sync + remove mock auth**로 정의됨 (`feat(api,web): better auth 1.6 integration + cookie sync + remove mock auth`). 다음 작업들은 W2-2의 직접 코드 변경에 의존하나, atomic commit 단위로 묶기에는:

- **결합도가 낮다** (별도 commit으로 분리해도 W2-2 검증 게이트에 영향 없음)
- **scope가 다르다** (코드 vs 문서, 자동화 vs 수동 절차)
- **타이밍이 늦다** (W2-2의 실제 commit 결과를 봐야 의사결정 가능)

따라서 W2-2.5 별도 wave로 묶어 처리한다. W2-2.5 commit은 **여러 개의 작은 atomic commit**으로 구성되며, 전체 wave는 1-day 이내에 완료한다.

#### 3.3.1 Husky pre-commit Node 22 PATH 자동화

**출처**: 현재 husky pre-commit hook (`pnpm lint-staged`)이 fresh subshell에서 nvm 환경을 inherit 못함. 매 commit 시 `export PATH="$HOME/.nvm/versions/node/v22.22.3/bin:$PATH"` 수동 prefix 필요.

**작업 내용**:

1. 옵션 A — `.husky/pre-commit` 수정:

   ```bash
   #!/usr/bin/env bash
   # Ensure Node 22 is on PATH for lint-staged (pnpm 11 requires Node 22+)
   if [ -f "$HOME/.nvm/nvm.sh" ]; then
     export NVM_DIR="$HOME/.nvm"
     . "$NVM_DIR/nvm.sh" --no-use
     nvm use 22 --silent || true
   fi
   exec pnpm lint-staged
   ```

2. 옵션 B — `.nvmrc` 파일 추가 (`v22.22.3`) + corepack 활용:


   그리고 husky가 `nvm use` 호출하도록

3. 옵션 C — engines-strict 활용 (`.npmrc`에 `engine-strict=true`)

**비교 + 결정**:
